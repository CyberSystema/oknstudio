Colophon · Reference Sheet

About this system.

A plain record of how OKN Studio is drawn, set, and assembled — typefaces, palette, components, and infrastructure. Read it as a reference, or as an appendix to the work. Everything visible on this site lives here, defined once.

01 · Preface

On the intent.

OKN Studio is the integrated workbench for the social team of the Orthodox Metropolis of Korea — not a product, not a platform, but a private workshop of four modules that fit together: analytics, a media pool, a data-ingest pipeline, and (forthcoming) a liturgical-editorial calendar.

The visual language is tech-forward by design. The team's work is digital; the infrastructure underneath is digital; the right aesthetic is one that respects that. A broadcast-engineering console meets a modern developer tool — dark, precise, legible, with a single confident accent.

Every module ships with the same vocabulary: a --signal mint-teal for interactive elements, a monospace for labels and paths, a geometric network mark that doubles as the brand. The result is a small surface area — one system, one voice, one aesthetic — so each instrument feels like part of the same studio.

All static HTML, served from Cloudflare's edge, with a handful of Functions behind for authentication and media proxy. No build step, no framework, no tracker. The entire site is smaller than most single hero images on the modern web.

05 · Stack

Quiet, verifiable infrastructure.

No black boxes. Everything OKN Studio touches runs on open protocols and auditable services — cheap to operate, fast to serve, durable to keep.

01 Runtime · Edge
Cloudflare Pages
Static HTML served from Cloudflare's global anycast edge. The only runtime that keeps both p95 latency and yearly cost near zero.
02 Serverless · Auth & API
Cloudflare Functions
HMAC-signed session cookies, media-proxy endpoints, and the ingest webhook — zero cold-start, co-located with the static.
03 Storage · Cold
Backblaze B2
EU-Central region, SSE-B2 at rest, lifecycle trimmed to current versions only. Bandwidth Alliance zeroes the egress.
04 CI/CD · Automation
GitHub Actions
Three workflows: analytics pipeline (weekly + push), deploy (on success), bucket-map (nightly to Drive).
05 Machine Learning
Python · sentence-transformers
Fourteen scikit-learn and transformer models, anchored on multilingual MiniLM-L12-v2 for 한국어 / English / Ἑλληνικά embedding.
06 Transport
rclone · Cyberduck
Operator-grade transfer tooling for the media archive and the weekly bucket-map upload to the team's Google Drive.
06 · Shortcuts

Keyboard shortcuts.

Most already familiar. Every module tries to behave like a professional tool: Esc closes, arrows navigate, slash focuses search.

S On the landing page — jumps to the Stack section.
/ In the Media Pool — focuses the filter field.
← → Inside the Media Pool preview — step between items.
Esc Closes any open preview, modal, or overlay.
Enter Submits the password on the authentication page.
Tab Walks focusable elements in reading order — everything's keyboard-reachable.
07 · Credits

Built, shipped.

Engineered and designed by Nikolaos Pinatsis, under his practice CyberSystema, from Larissa in central Greece. The work serves the Orthodox Metropolis of Korea and its small media team in Seoul.

The typography relies on Sora by Rasmus Andersson, IBM Plex Sans and IBM Plex Mono by IBM, and Noto Sans KR by Google's Noto project. Every typeface is SIL OFL — thanks are owed to their makers.

The broader stack — Cloudflare Pages, Backblaze B2, GitHub Actions, scikit-learn, sentence-transformers, rclone, Cyberduck — is open, well-documented, and built by teams working in public. The shoulders are large and the debt is acknowledged.
